2.数据库设计
抽取实体:用户登录信息,日志,回复,主人回复,用户详情信息
分析其中的属性
用户登陆信息:账号,密码,头像,名称
用户详情信息:真实姓名,星座,血型...
日志:内容,标题,日期,作者
回复:内容,日期,作者,日志
主人回复:内容,日期,作者,回复
分析实体之间的关系
用户登录信息:用户详情信息 1:1 PK
用户和日志之间的关系,一个用户会拥有多个日志
日志和回复之间也是一对多关系 1:N
回复和主人回复之间, 是 1:1 UK
用户:好友 M:N

数据库范式:
第一范式:列不可再分
第二范式:一张表只表达一个含义(只描述一件事情)
第三范式:表中的每一列和主键都是直接依赖关系,而不是间接依赖
4.数据库设计的范式和数据库的查询性能很多时候是相悖的,我们需要根据业务做一个选择
-查询频次不高的情况下,我们更倾向于设计范式,从而提高存储效率
-反之查询频次较高,我们更愿意降低规范度,允许特定的冗余,从而提高查询性能
